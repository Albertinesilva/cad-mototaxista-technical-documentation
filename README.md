# 🏍️ CAD-MOTOTAXISTA

## Documentação Técnica – Sistema de Gestão Pública

---

## 🚀 Visão Geral

O **CAD-MOTOTAXISTA** é um sistema web desenvolvido como Trabalho de Conclusão de Curso (TCC) no curso de **Análise e Desenvolvimento de Sistemas do Instituto Federal da Bahia (IFBA)**.

O projeto surgiu a partir de uma demanda real da **Secretaria Municipal de Trânsito e Transporte (SMTT) de Santo Antônio de Jesus (BA)**, que não possuía um sistema informatizado para gestão e controle de mototaxistas e motofretistas.

A solução foi concebida para estruturar, digitalizar e organizar processos administrativos relacionados a:

- Cadastro de profissionais
- Emissão e controle de alvarás
- Regularização documental
- Fiscalização administrativa
- Controle de vencimentos e validações
- Organização e rastreabilidade de informações

O desenvolvimento foi orientado por princípios de engenharia de software, organização arquitetural e aplicabilidade em ambiente institucional.

---

## 🏛️ Problema Resolvido

Antes da implementação do sistema:

- Processos predominantemente manuais
- Controle documental descentralizado
- Dificuldade de fiscalização
- Baixa rastreabilidade de informações

Com o CAD-MOTOTAXISTA:

- Centralização estruturada dos dados
- Modelagem relacional consistente
- Organização das regras de negócio
- Base sólida para apoio à decisão administrativa

## 🧠 Arquitetura e Engenharia

O sistema foi desenvolvido como uma aplicação web monolítica server-side utilizando Spring Boot e Thymeleaf, seguindo o padrão arquitetural MVC em camadas.

A estrutura foi projetada com foco em separação de responsabilidades, segurança, controle de acesso e manutenibilidade.

O backend foi organizado em camadas bem definidas (Controller, Service e Repository), contemplando:

- Separação clara entre lógica de apresentação, regras de negócio e persistência
- Organização modular de componentes
- Padronização de nomenclaturas
- Validações estruturadas no backend
- Modelagem relacional normalizada
- Controle explícito de regras de negócio
- Documentação técnica rastreável

A documentação aqui apresentada permite compreender as decisões arquiteturais, as tecnologias adotadas e os fundamentos técnicos que sustentam a solução.
